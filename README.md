# PHPTravels Web Application – End-to-End Manual Testing Project

## Project Overview
This project demonstrates manual functional testing performed on the PHPTravels web application.

- Application type: Travel booking web application  
- Testing type: Manual functional testing  
- Focus areas: User journeys, validations, error handling, and authorization checks  

Testing was performed on frontend user-facing features with emphasis on real-world end-to-end flows.

---

## Scope of Testing

### ✔ In Scope
- User Signup, Login, Logout, and Forgot Password  
- Flight Search and Search Results  
- Modify Search and Filters  
- Flight Booking Flow (Guest and Logged-in User)  
- Payment Gateway Redirection  
- Booking Confirmation and Invoice  
- Booking Cancellation Request  
- User Dashboard and My Bookings  
- User Profile Management  
- Header and Footer Navigation  

### ✖ Out of Scope
- Hotel, Tours, Cars, and Visa modules  
- Admin, Agent, Supplier, and Vendor functionalities  
- Backend and database testing  
- Performance, load, and stress testing  
- Security and penetration testing  
- Actual payment transaction processing

## Test Approach
Testing was carried out by analyzing application behavior, designing test scenarios, executing positive and negative test cases, and logging defects based on deviations between expected and actual results. End-to-end user workflows were validated from a customer perspective.
